Hello!

I don't know why, but the currently provided files glyph-list-1.000,
glyph-list-1.001, and glyph-list-2.000 cannot be reproduced using the
script generate-glyph-list.sh, maybe those glyph files aren't up to
date with the generate-glyph-list.sh and need recreation.

More important is the fact that this seems to be the cause, according
to my experience, of the invalid vector<T> subscript error with
MinionPro and Yap! I have just created new glyph files using the
current generate-glyph-list.sh and recompiled MinionPro using the
--pack option point to the new glyph file and surprise: the Yap error
is gone forever!

Maybe someone else can confirm (reproduce) this report...
