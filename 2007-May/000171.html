Hello,

when I compile a document with MinionPro package and the [medfamily] 
option, the following problem occurs: With osf, tabular figures and with 
lining, proportional figures, LaTeX produces only crossed boxes instead 
of the correct glyphs with the semibold font. Strangely, the 
combinations (osf, proportional) and (lining, tabular) work, so both the 
osf and the lining glyphs can be displayed, in principle.

I have the new release v2.1 of MinionPro installed under MikTeX. The 
MinionPro fonts come from Adobe Illustrator CS2, where the normal, bold 
and semibold weights are supplied (font version 2.000 if I remember 
correctly).

Can someone reproduce this bug? Or even better, does anyone know a 
solution? I attach a minimal example file below.

Apart from this little bug (which can certainly be fixed) I would like 
to thank the authors of the MinionPro package. You did a great job, 
since it's one of the most usable LaTeX font packages and one of the 
very few which one can seriously typeset mathematics with.

Regards,

Johann

\documentclass{minimal}
\usepackage[medfamily]{MinionPro}
\def\sample{\figureversion{lf}\figureversion{tab}lining, tabular: 123,
  \figureversion{osf}text, tabular: 123,
  \figureversion{lf}\figureversion{prop}lining, proportional: 123,
  \figureversion{osf}text, proportional: 123
}
\begin{document}
\sample

\fontseries{b}\selectfont\sample

\fontseries{eb}\selectfont\sample
\end{document}
