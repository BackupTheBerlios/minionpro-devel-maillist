Hi Achim,

thanks for replying.   I've pasted the output below.  Meanwhile, just  
a couple more things that I've learned, which may or may not be  
relevant.  First, when I emailed earlier, I hadn't realized that the / 
usr/local/teTeX hierarchy is irrelevant to my distribution of gwTeX  
(which is the most recent one, based on TeXLive 2007.)  I think it's  
left over on the machine from an earlier TeX installation.  Second:   
I actually have two machines.  On one, I got MinionPro to work, on  
the other I've failed.  And I've noticed a few differences between  
these machines.  First, when I run updmap (see the output below), on  
each machine it says that the config file it's using is /usr/local/ 
gwTeX/texmf.local/web2c/updmap.cfg.   But when I look inside this  
file, it has Map MinionPro.map on the machine in which things are NOT  
working.  On the machine where the font IS working, /usr/local/gwTeX/ 
texmf.local/web2c/updmap.cfg does NOT contain this command.   Second,  
I noticed that the .tfm files on the computer where everything is  
working are different from the files on the computer where it isn't  
working.  The nonworking computer has, e.g., MinionPro-Regular-Base- 
aa.tfm; whereas the working computer has, e.g., MinionPro-Regular-lf- 
l1-lgi--base.tfm.  I also note that the .tfm files on the working  
computer were created in 2005, whereas the .tfm files on the  
nonworking computer were created by me recently.  (I installed  
MinionPro on the working computer a few months ago; could this have  
something to do with the fact that the package was recently updated?)
