Hi Andreas,

Thanks for this bug report, which is fixed in 2.62.  Simple fix: I 
downgraded the "--unicoding command" message from an error to a warning.

2.62 also has a more significant fix.  The t1dotlessj program previously 
generated fonts with bad UniqueID values, the same as the input font. 
This can produce some real problems with older printers.  The new 
version is smarter.  I'd recommend using it.

Eddie
