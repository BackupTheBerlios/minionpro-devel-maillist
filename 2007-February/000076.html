Hi, the following question is related to both MnSymbol and MinionPro:

The MnSymbol package and the MinionPro package redeclare some symbol  
fonts with different encodings:
symbols as U (standard LaTeX: OMS)
operators as T1 (standard LaTeX: OT1).

For this, LaTeX issues a font warning:

> LaTeX Font Warning: Encoding `OMS' has changed to `U' for symbol font
> (Font)              `symbols' in the math version `normal' on input  
> line 120.
>
(this one is caused by MnSymbol.sty) and

> LaTeX Font Warning: Encoding `OT1' has changed to `T1' for symbol font
> (Font)              `operators' in the math version `normal' on  
> input line 111.
>
(caused by MinionPro.sty).

The respective lines in the .sty files are
   \DeclareSymbolFont{symbols}{U}{MnSymbolF}{m}{n}            (in  
MnSymbol.sty)
and
   \DeclareSymbolFont{operators}{T1} {\Mn at Math@Family}{m}{n}  (in  
MinionPro.sty)

First, does this warning indicate that something might actually go  
wrong?
What do I have to do to get rid of this warning? (It is rather  
annoying and might distract from actual problems.)

I am using MnSymbol 1.4 and MinionPro 2.0
